# MainSubKiosk

키오스크 전용 이미지 갤러리 앱 — 버튼을 누르면 상세 페이지가 팝업으로 열리는 터치 기반 디지털 안내 시스템

---

## 📋 프로젝트 개요

| 항목             | 내용                                          |
| ---------------- | --------------------------------------------- |
| **엔진**   | Unity (C#) 6000.3.9f1                         |
| **용도**   | 키오스크/전시 디스플레이용 이미지 갤러리      |
| **해상도** | Settings.txt에서 변경 가능 (기본: 1920×1080) |
| **프레임** | 60FPS 제한 (장시간 안정 구동 최적화)          |

## ⚙️ Settings.txt 설정 항목

> **빌드(exe) 모드:** exe 파일과 같은 폴더에 `Settings.txt` 배치

| 키                           | 기본값                    | 설명                                                                                                     |
| ---------------------------- | ------------------------- | -------------------------------------------------------------------------------------------------------- |
| `Idle_Timeout`             | `60`                    | 터치 없을 시 메인 화면으로 복귀하는 시간 (초)                                                            |
| `Transition_Speed`         | `0.5`                   | 화면 전환 애니메이션 속도 (초)                                                                           |
| `Resolution`               | `1920x1080`             | 프로그램 실행 해상도                                                                                     |
| `Target_Frame_Rate`        | `60`                    | 최대 프레임 속도 제한 (높을수록 부드럽지만 부하 증가)                                                    |
| `Show_Error_Popup`         | `true`                  | 시작 시 이미지 누락 경고 팝업 표시 여부                                                                  |
| `Show_Runtime_Error_Popup` | `true`                  | 사용 중 에러 발생 시 팝업 표시 여부                                                                      |
| `Hide_Mouse_Cursor`        | `true`                  | 마우스 커서를 화면에 표시할지 숨길지 여부 (터치는 정상 작동)                                             |
| `Image_Extensions`         | `.png;.jpg;.jpeg`       | 인식할 이미지 확장자 (`;` 구분)                                                                        |
| `Exclude_Grid_Buttons`     | `0000-btn`              | 주요 법령 버튼설정(주요법령의 버튼이름을 넣으면 됩니다.) : 그리드에 자동 생성하지 않을 버튼 (`;` 구분) |
| `Required_Buttons`         | `0000-btn;0001-btn;...` | 반드시 존재해야 하는 필수 버튼 (`;` 구분, 비우면 검사 안 함)                                           |

## 🖼️ 이미지 파일 규칙

콘텐츠 폴더(`Content_MainSubKiosk/`)에 아래 규칙대로 파일을 넣으면 자동 인식됩니다:

| 파일명 패턴       | 역할                            | 예시              |
| ----------------- | ------------------------------- | ----------------- |
| `XXXX-btn.png`  | 메인 화면의 버튼 이미지         | `0001-btn.png`  |
| `XXXX-page.png` | 버튼 클릭 시 열리는 상세 페이지 | `0001-page.png` |

- **btn과 page는 반드시 쌍**으로 존재해야 합니다 (예: `0001-btn` ↔ `0001-page`)
- 대소문자를 구분하지 않습니다 (`btn` = `BTN` = `Btn`)
- 파일을 추가/삭제하면 다음 실행 시 자동 반영됩니다

## 📝 로그 시스템 (Logger)

앱 실행 중 발생하는 **경고(Warning)**, **에러(Error)**, **예외(Exception)**는 `Logger.cs` 시스템을 통해 백그라운드에서 자동으로 파일에 저장됩니다. 이를 통해 현장에 배포된 기기에서 문제가 발생해도 사후 분석이 가능합니다.

- **저장 위치:** 윈도우 기본 경로 `%USERPROFILE%\AppData\LocalLow\MetaDevs\MainSubKiosk\Logs\`
  (유니티의 `Application.persistentDataPath`를 따릅니다)
- **파일 이름 형식:** `YYMMDD-HHMMSS.log` (앱이 실행될 때마다 새 파일로 생성)
- **저장 용량 관리:** 최대 보관 개수(기본값: 가장 최근 1000개 파일)를 초과하면 가장 오래된 파일부터 자동 삭제하여 디스크 용량을 관리합니다.

## 🚀 시작하기

### 1. 콘텐츠 준비

```
Assets/StreamingAssets/Content_MainSubKiosk/
├── 0000-btn.png    ← 주요법령 버튼 : 그리드 외부 전용 버튼
```

### 4. 빌드

빌드 후 `Settings.txt`를 exe 파일과 같은 폴더에 배치

## 🔧 그리드 외부 수동 버튼 설정

그리드 밖에 별도로 배치된 버튼(예: 주요법령 버튼)은 **Settings.txt만으로 관리**됩니다:

1. `Settings.txt`의 `Exclude_Grid_Buttons`에 해당 버튼 키를 등록 (예: `0000-btn`)
2. 스크립트가 자동으로 해당 키의 이미지를 로딩하고 페이지 팝업 이벤트를 연결

>  별도로 키를 입력할 필요 없이, Settings.txt 에서 관리됩니다.
