# 🖥️ MainSubKiosk — 키오스크 관리 매뉴얼

> 프로그램을 직접 수정할 필요 없이, **이미지 파일 교체**와 **메모장(Settings.txt) 수정**만으로 키오스크 화면을 자유롭게 바꿀 수 있습니다.

---

## 📋 프로그램 소개

| 항목                  | 내용                                          |
| :-------------------- | :-------------------------------------------- |
| **프로그램명**  | MainSubKiosk                                  |
| **엔진**        | Unity 6000.3.9f1                              |
| **용도**        | 키오스크 / 전시 디스플레이용 이미지 갤러리    |
| **화면 해상도** | Settings.txt에서 변경 가능 (기본: 1920×1080) |
| **화면 프레임** | 60FPS 제한 (장시간 안정 구동 최적화)          |

---

## 1장. 프로그램 시작하기

프로그램이 켜지면 아래 순서로 자동 작동합니다:

1. **파일 점검** — 필요한 이미지 파일이 전부 있는지, 깨지지 않았는지 자동으로 검사합니다.
2. **메인 화면 표시** — 검사가 끝나면 메인 화면에 버튼들이 나타납니다.
3. **자동 복귀** — 손님이 화면을 일정 시간(기본 60초) 동안 만지지 않으면 자동으로 메인 화면으로 돌아갑니다.

> ⚠️ 만약 프로그램이 켜지자마자 **빨간색 에러 팝업창**이 뜬다면, 이 문서의 **4장. 문제 해결**을 먼저 읽어주세요.

## 2장. 📺 이미지 교체 가이드 (가장 중요!)

### 2-1. 작업할 폴더 위치

이미지 파일들은 아래 폴더에 들어있습니다:

```
프로그램 폴더/
└── MainSubKiosk_Data/
    └── StreamingAssets/
        └── Content_MainSubKiosk/   ←  이 폴더를 열어주세요!
            ├── (버튼, 배경, 탭 이미지들)
            └── Image/              ←  슬라이드 사진 폴더
```

> 💡 **작업 순서**: 파일 탐색기(윈도우 폴더)에서 `Content_MainSubKiosk` 폴더를 열고, 아래 설명에 맞는 이름으로 이미지를 넣어주면 됩니다.

---

### 2-2. 파일 이름 규칙 총정리

이 프로그램은 **파일 이름**을 보고 어떤 화면에 어떤 그림을 띄울지 알아냅니다. 따라서 **이름을 정확히** 맞춰주셔야 합니다.

#### 📌 일반 버튼 (1번~6번) 파일 목록

1번 버튼을 예로 들면, 아래 **5종류**의 이미지 파일이 한 세트입니다:

| 파일 이름             | 뜻               | 설명                                                   |
| :-------------------- | :--------------- | :----------------------------------------------------- |
| `0001_btn_off.png`  | 버튼 (평상시)    | 메인 화면에서 손님이 아직 누르지 않은 상태의 버튼 모양 |
| `0001_btn_on.png`   | 버튼 (눌렀을 때) | 메인 화면에서 손님이 버튼을 눌렀을 때 바뀌는 모양      |
| `0001_page.png`     | 상세 화면 배경   | 버튼을 눌러 들어간 상세 화면의 뒷 배경                 |
| `0001_page_off.png` | 하단 탭 (평상시) | 상세 화면 아래쪽에 나오는 번호 탭의 평소 모양          |
| `0001_page_on.png`  | 하단 탭 (선택됨) | 현재 보고 있는 번호의 탭이 강조되어 보이는 모양        |

> 2번은 `0002_btn_off.png`, `0002_btn_on.png`, ... 식으로 **앞 4자리 숫자만 바꿔주시면** 됩니다. (3번~6번도 동일)

#### 📌 특수 버튼 (0000번) 파일 목록

0000번은 일반 버튼과 약간 다릅니다. on/off가 없는 단독 버튼입니다.

| 파일 이름          | 설명                                     |
| :----------------- | :--------------------------------------- |
| `0000_btn.png`   | 메인 화면에 표시되는 특수 버튼 모양      |
| `0000_page.png`  | 특수 버튼을 눌렀을 때 나오는 페이지 배경 |
| `0000_close.png` | 특수 페이지의 닫기(X) 버튼 모양          |

---

### 2-3. 슬라이드 사진 넣기 (Image 폴더)

상세 화면 안에서 자동으로 넘어가는 **슬라이드 사진**들은 `Image` 폴더 안에 넣습니다.

```
Content_MainSubKiosk/
└── Image/
    ├── 1_1.png      ← 1번 상세화면의 사진 1장째
    ├── 1_2.png      ← 1번 상세화면의 사진 2장째
    ├── 1_3.jpg      ← 1번 상세화면의 사진 3장째
    ├── 2_1.png      ← 2번 상세화면의 사진 1장째
    ├── 2_2.png      ← 2번 상세화면의 사진 2장째
    └── ...
```

**이름 규칙:** `(버튼번호)_(순서번호).png` 또는 `.jpg`

- 앞 숫자 = 메인 버튼 번호
- 뒤 숫자 = 사진 순서 (1부터 시작)
- 앞 숫자에 0을 붙여도 되고 안 붙여도 됩니다 (자동 인식)

| 형식          | 예시                           | 설명                          |
| :------------ | :----------------------------- | :---------------------------- |
| 짧은 형식     | `1_1.png`, `6_2.png`       | 앞의 0 없이 번호만 사용       |
| 제로패딩 형식 | `0001_1.png`, `0006_2.png` | 버튼 번호와 동일한 4자리 형식 |

> 💡 두 형식을 섞어서 사용해도 됩니다. 예) `1_1.png`과 `0001_2.png`가 같은 폴더에 있어도 둘 다 1번 슬라이드로 인식됩니다.

> 🚨 **필수 조건**: 버튼이 존재하면 해당 번호의 슬라이드 사진이 **최소 1장** 이상 있어야 합니다!
> 예) 1번 버튼이 있는데 `Image` 폴더에 `1_` 또는 `0001_`로 시작하는 사진이 하나도 없으면, 프로그램이 켜질 때 에러 팝업이 뜹니다.

> 💡 사진이 딱 1장만 있으면, 자동 넘김과 좌우 화살표 버튼은 자동으로 사라집니다.
> 여러 장이면 3초마다 자동으로 넘어가며, 마지막 사진에서 다음을 누르면 첫 사진으로 돌아갑니다.

### 2-4. 전체 폴더 구조 한눈에 보기

```
Content_MainSubKiosk/
```

## 3장. ⚙️ 키오스크 환경 설정 (Settings.txt)

프로그램 실행 파일(exe) 옆에 있는 `Settings.txt` 파일을 **메모장**으로 열면, 키오스크의 동작을 바꿀 수 있습니다.

### 3-1. 작성 방법

한 줄에 하나씩, **설정 이름**과 **값**을 쉼표(`,`)로 구분하여 적습니다.

```
Idle_Timeout, 60
Target_Frame_Rate, 60
Resolution, 1920x1080
```

> 💡 줄 앞에 `#`을 붙이면 그 줄은 무시됩니다. (메모 남기기 용도)

### 3-2. 설정 항목 전체 목록

| 설정 이름                    | 설명                                                                                               |       기본값       | 예시                                       |
| :--------------------------- | :------------------------------------------------------------------------------------------------- | :-----------------: | :----------------------------------------- |
| `Idle_Timeout`             | 손님이 화면을 만지지 않으면 자동으로 메인 화면으로 돌아가는 시간 (초 단위)                         |       `60`       | `Idle_Timeout, 120` → 2분               |
| `Target_Frame_Rate`        | 화면의 초당 프레임 수 제한 (숫자가 낮을수록 전력 절약)                                             |       `60`       | `Target_Frame_Rate, 30` → 30프레임      |
| `Transition_Speed`         | 화면이 전환되는 애니메이션 속도 (초 단위, 작을수록 빠름)                                           |       `0.5`       | `Transition_Speed, 0.3` → 더 빠르게     |
| `Resolution`               | 키오스크 화면 해상도                                                                               |    `1920x1080`    | `Resolution, 720x1280` → 세로형         |
| `Required_Buttons`         | 프로그램 시작 시 반드시 존재해야 하는 버튼 번호. 없으면 에러 팝업이 뜹니다.`~`로 범위 지정 가능. |       (없음)       | `Required_Buttons, 0001~0006`            |
| `Exclude_Grid_Buttons`     | 메인 화면의 격자(그리드) 안에 넣지 않고, 화면 구석에 따로 고정 배치할 버튼 번호                    |       (없음)       | `Exclude_Grid_Buttons, 0000`             |
| `Hide_Mouse_Cursor`        | 마우스 커서(화살표)를 화면에서 숨길지 여부 (터치는 정상 작동)                                      |      `true`      | `Hide_Mouse_Cursor, false` → 커서 보임  |
| `Show_Error_Popup`         | 프로그램 시작 시 에러 팝업을 띄울지 여부                                                           |      `true`      | `Show_Error_Popup, false` → 팝업 끔     |
| `Show_Runtime_Error_Popup` | 프로그램 사용 도중 에러 팝업을 띄울지 여부                                                         |      `true`      |                                            |
| `Image_Extensions`         | 프로그램이 인식하는 이미지 파일 확장자 (세미콜론으로 구분)                                         | `.png;.jpg;.jpeg` | `Image_Extensions, .png;.jpg;.jpeg;.bmp` |

## 4장. 🚨 문제 해결 (에러 팝업창 대처법)

키오스크를 켰을 때 화면에 **빨간색 에러 팝업창**이 뜨더라도 당황하지 마세요!
아래 목록에서 해당하는 문구를 찾아 조치하시면 됩니다.

### 에러 1. "~파일을 찾을 수 없습니다"

**원인**: 이미지 파일이 `Content_MainSubKiosk` 폴더에 없거나, 파일 이름에 오타가 있습니다.

**해결 방법**:

1. `Content_MainSubKiosk` 폴더를 열어주세요.
2. 에러 메시지에 적힌 파일 이름(예: `0001_btn_off`)이 폴더 안에 있는지 확인해주세요.
3. 없다면 USB에서 해당 파일을 복사해 넣어주세요.
4. 있는데도 에러가 뜬다면, **파일 이름에 띄어쓰기나 오타**가 없는지 다시 확인해주세요.

### 에러 2. "~파일 내용이 텅 비어있습니다 (0바이트)"

**원인**: 파일이 존재는 하지만 내용이 비어있습니다. USB에서 복사하다가 파일이 깨졌을 가능성이 높습니다.

**해결 방법**:

1. 해당 파일을 삭제합니다.
2. 원본 USB(또는 백업 폴더)에서 같은 이름의 파일을 다시 복사합니다.
3. 복사한 파일을 마우스 오른쪽 클릭 → `속성`에서 **파일 크기가 0이 아닌지** 확인합니다.

### 에러 3. "~슬라이드 사진이 단 1장도 없습니다"

**원인**: 메인 화면에 버튼은 있는데, 그 번호에 해당하는 **슬라이드 사진**이 `Image` 폴더에 없습니다.

**해결 방법**:

1. `Content_MainSubKiosk/Image/` 폴더를 엽니다.
2. 에러 메시지에 적힌 번호(예: `'1'번`)에 맞는 사진 파일을 넣어주세요.
3. 파일 이름 예시: `1_1.png` 또는 `0001_1.png` (1번 버튼용 첫 번째 사진)

### 에러 4. "~버튼은 존재하는데, ~page 파일은 누락되었습니다"

**원인**: 버튼 그림은 있지만 상세 화면 배경 그림이 없습니다.

**해결 방법**:

- `Content_MainSubKiosk` 폴더에 해당 `_page.png` 파일을 넣어주세요.
- 예: `0001_btn_off.png`는 있는데 `0001_page.png`가 없을 때 → `0001_page.png`를 넣으면 해결.
