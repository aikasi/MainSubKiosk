# Media Table (Secondary Application) 개발  가이드

이 문서는 기존 `MainSubKiosk` 빌드(README.md)와 완전히 분리되어 독립적으로 작동하는 **듀얼 모니터용 전자책(Media Table) 애플리케이션**에 대한 설명서입니다. 

현재 스와이프와 버튼식 두가지가 있지만 버튼식을 선택했습니다.

스와이프 형식은 제공된 에셋에서는 **`Input.mousePosition`** 을 사용하여 매 프레임마다 마우스를 추적하는 형태로 만들어져있어서 두 모니터에서 각각 동시에 터치하였을 때 발생할 수 있는 마우스 좌표 순간이동 버그가 발생할 수 있습니다.


---

## 🏗️ 1. 아키텍처 (Architecture)

Media Table은 듀얼 디스플레이에서 각기 다른 전자책을 독립적으로 열고 넘길 수 있도록 설계되었습니다. 메모리 최적화를 위해 싱글톤(스캐너) 시스템과 비동기(Async) 객체 생성 패턴을 혼합 사용합니다.

### 핵심 스크립트 역할 분담

* **`BookPageScanner.cs` (전역/싱글톤):**
  * 앱 실행 시 `Content_MediaTable` 폴더를 **단 1번**만 스캔합니다.
  * "-btn" 문자가 들어간 썸네일과 "-page_x" 문자가 들어간 책 속지를 분류해 메모리 주소(캐시)만 들고 있는 **도서관 사서** 역할입니다.
  * *Natural Sort(자연 정렬)* 가 적용되어 `page_1`, `page_10`, `page_2` 순서 상관없이 숫자의 크기대로 **항상 올바르게 넘버링이 정렬**됩니다.
* **`MediaMainScreenManager.cs` (각 캔버스당 1개):**
  * 화면에 처음 뜨는 썸네일 버튼 그리드를 생성합니다.
  * 속도를 위해 `Task.WhenAll` 병렬 로딩 처리로 화면에 띄웁니다.
* **`MediaBookManager.cs` (각 캔버스당 1개):**
  * 썸네일을 클릭하면 실제로 책(3D Asset)을 팝업시키는 **지휘관**입니다.
  * 극단적인 광클(연타 버그) 시 팝업이 여러 개 뜨며 에러를 뿜는 현상을 막기 위해 철통 락(`isOpening`)이 쳐져 있습니다.
  * 스와이프 및 화살표 양방향 입력 시 안전하게 다음 장이 준비(await) 되는 것을 기다렸다가 책을 넘깁니다.
* **`PageTexturePool.cs` (각 캔버스당 1개):**
  * Book 에셋이 수집한 이미지를 RAM에 모두 올리면 PC가 멈추거나 다운될 수 있으므로, **현재 시청 중인 5~6장의 페이지만 메모리에 살려두고 나머지는 강제 파기(Destroy)**하는 '가비지 컬렉터' 겸 '메모리 방어벽'입니다.

## 📂 2. 폴더 구성 및 파일 네이밍 규칙 (.png / .jpg)

콘텐츠들은 빌드 폴더의 `StreamingAssets\Content_MediaTable` 안에 넣어야 앱이 인식합니다. (폴더명은 Settings.txt에서 변경 가능)

### 2-1. 필수 이름 규칙

반드시 아이템 고유 ID(ex: `0001`, `event_A` 등)를 앞에 적고 대시(-)를 붙여야 합니다.

언제든 바뀔수 있습니다.( 아직 미정 )

* **썸네일 버튼 이미지:** `[고유ID]-btn.png`
  * *(예시)* `0001-btn.png`, `0002-btn.jpg`
* **책 속지 이미지:** `[고유ID]-page_[숫자].png`
  * *(예시)* `0001-page_1.png` (가장 처음 보이는 왼쪽 커버)
  * *(예시)* `0001-page_2.png` (가장 처음 보이는 오른쪽 커버)
  * *(예시)* `0001-page_10.png` (숫자가 10 이상 넘어가도 오작동 없이 정확히 맨 뒤로 정렬됩니다)
* **단일 페이지 (1페이지만 있을 경우 예외 허용):** `[고유ID]-page.png`

> ⚠️ 주의: 어떤 ID에 대해 `-btn` 이미지 파일만 넣고, 정작 안에 보여줄 `-page_x` 속지 파일이 한 장도 없는 빈 깡통 상태라면 썸네일 버튼 생성을 스킵하고 **화면에 ErrorPopup 경고창을 띄워 관리자에게 오류를 알립니다.**

## ⚙️ 3. Settings.txt 전용 옵션

`StreamingAssets/Settings.txt` 파일을 메모장으로 열고 아래의 수치를 편집하여 Kiosk를 세팅합니다. MediaTable에 적용되는 전용 설정들입니다.

**주요 옵션표:**

| 항목명                      | 기본값             | 설명                                                                                                                                                                                                                                                                      |
| :-------------------------- | :----------------- | :------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| `MediaTable_Btn_Path`     | Content_MediaTable | 썸네일을 스캔할 기준 디렉토리 이름입니다.                                                                                                                                                                                                                                 |
| `MediaTable_Page_Path`    | Content_MediaTable | 책 속지를 스캔할 기준 디렉토리 이름입니다.                                                                                                                                                                                                                                |
| `Keep_Previous_Pages`     | 2                  | 책장을 뒤로 넘길 때**과거의 앞장 이미지를 메모리에 몇 장이나 남겨둘 것인지** 결정합니다.`<br>`이 숫자가 클수록 뒷장으로 넘어갈 때 딜레이가 0에 가까워지지만 RAM 소모가 늘어납니다.                                                                                |
| `Transition_Speed`        | 0.5                | (초 단위) 썸네일을 눌러서 책 상세 화면(팝업 창)이 애니메이션으로 스르륵 켜지거나 꺼지는 데 걸리는 시간입니다.                                                                                                                                                             |
| `Show_Missing_Page_Error` | true               | `true`일 경우, 폴더 안에 `page_1, page_2, page_10` 과 같이 중간 번호(3~9)가 누락된 파일 구조가 발견되면 스캐너가 즉시 화면 정중앙에 빨간 경고창(Error Popup)을 띄웁니다.`<br>` `false`로 설정 시 에러 창은 뜨지 않지만 유니티 콘솔(배경 Log)에는 기록이 남습니다. |

## 🚀 4. 에디터 듀얼 캔버스 설정법 (작업자 참고용)

Phase 3 듀얼 디스플레이 송출을 위해 씬(Scene)을 분리할 때의 가이드입니다.

1. `Display1_Canvas` (왼쪽 모니터용)에 `MediaMainScreenManager` 및 `MediaBookManager`를 세팅 완료합니다.
2. `Display1_Canvas`를 복제(Ctrl+D)하여 `Display2_Canvas`로 이름을 변경합니다.
3. `Display2_Canvas`의 **Target Display 속성을 Display 2**로 변경합니다.
4. `Display2_Canvas` 안에 있는 `MediaBookManager`의 Inspector를 열고, 반드시 **`Display2_Canvas`의 자식들(자신의 캔버스 내부 오브젝트)**로 모든 Target(AutoFlip, DetailPanel, MemoryPool 등)들을 새롭게 드래그 앤 드롭해서 **교체 연동**해 줍니다.
   *(이 과정을 생략하면 2번 디스플레이에서 클릭했는데 1번 디스플레이의 책이 펴지는 대참사가 일어납니다)*
5. 전체 씬에서 `BookPageScanner`(스캐너) 컴포넌트가 **딱 하나**만 존재하는지 최종 확인합니다.

이 단계를 모두 거치면 완전 독립 구동형 듀얼 테이블 키오스크 세팅이 완료됩니다.
